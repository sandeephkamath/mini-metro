## Confirmed correct (no findings) ‚ÄĒ free-form exploration

- **Pinch starting mid single-finger drag**: starting a line-draw drag (touch down on a station, move partway toward a second station), then adding a second finger before releasing. The code (`useMouseInput.ts` `handleTouchStart`) ends the in-flight single-touch drag at the second finger's touch-down moment (equivalent to a mouse-up at that mid-air point, which is not a valid station, so the drag is cancelled per `core/logic.md` ¬ß4 "Releasing anywhere other than a valid Node cancels the drag"), then begins the pinch normally. No line was accidentally created, no page errors.
- **Dragging the finger away from a Line swatch during a hold, without lifting** (single swatch, single finger): confirmed the hold still completes and deletes the Line after 600ms+ even though the finger physically moved far from the swatch's DOM element mid-hold, and no page errors occurred. This matches `themes/metro.md` ¬ß6's explicit note that touch intentionally has no `mouseleave` equivalent for this gesture ("touch uses touchstart/touchend/touchcancel in place of mousedown/mouseup/mouseleave") ‚ÄĒ touch-capture semantics keep `touchend` bound to the original element regardless of where the finger ends up, so this is working as designed, not a bug.
- **Touch input while the Weekly Upgrade choice popup is open**: reached the popup live (drew lines, used debug mode + 4x speed + paused auto-spawns to survive to the Milestone Event without an active player), then attempted a touch line-draw drag directly over the canvas underneath the modal. The drag had no effect (no line was created ‚ÄĒ canvas pixel at the intended segment stayed background-colored), confirming the modal's full-cover overlay correctly intercepts touch input and the game clock/canvas are untouched while the choice is pending. Resolving the choice via a real touch tap on an option (not a mouse click) worked correctly and un-paused the game. No bug.
- **Viewport resize (orientation change) mid-game**: resized the page from the iPhone 13 portrait viewport (390√ó664) to a landscape-shaped viewport (844√ó390) mid-session. The canvas/HUD stage rescaled to fit with no horizontal or vertical page overflow, and a touch-drawn line immediately after the resize was drawn correctly at the new scale. Resizing back to portrait also worked cleanly. No page errors, no bug.
- **Touch in the letterboxed dead zone** (viewport area outside the scaled canvas, e.g. above it in portrait mode where the stage doesn't fill the viewport height): touching and dragging in that area produced no page errors and had no effect on the game (as expected ‚ÄĒ nothing interactive lives there).
- **Touch drag starting/ending at the extreme logical corners of the canvas** (near (0,0) and (799,599)): no errors, no coordinate-math edge-case crashes.
- **Extremely rapid repeated pinch in/out** (30 iterations of pinch-out-then-pinch-in with no settle time between direction reversals, tighter than the fixed `touch-pinch-zoom.spec.ts` stress test): no page errors, final screenshot renders cleanly with all three stations visible near the zoomed-out center, consistent with `CAMERA_MIN_ZOOM` clamping. No bug.

## Notes for whoever picks this up

- M1 only requires two simultaneous touch points on two different DOM elements (the Line swatches), which is unreachable via mouse (a single pointer can't `mousedown` on two elements at once) ‚ÄĒ it's a genuinely mobile/multi-touch-only regression, which is why the existing desktop-only test coverage never caught it. Recommended fix if picked up: make `holdingLineId`/`holdTimerRef` per-Line (e.g. a `Map<string, number>` of pending timers, or one `useRef`/`useState` pair per swatch) instead of a single shared value, so concurrent holds on different Lines don't interfere with each other's timer lifecycle.
- Did not find any issues with the core Responsive Fit / Touch Draw / Touch Pan / Pinch Zoom / Two-Finger Pan / Touch Hold-to-Delete flows themselves ‚ÄĒ all behave per `themes/metro.md` ¬ß6 and the mobile flow table in `specs/testing.md` ¬ß3.1.
